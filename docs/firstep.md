# Firstep - 사이드프로젝트 런치패드

> "두루뭉실한 아이디어만 있고 어디서 시작해야 할지 몰랐던 나를 위해 만들었습니다. Firstep은 AI가 아이디어를 단계별 개발 로드맵으로 만들어주고, 완료까지 옆에서 가이드합니다."

## 서비스 소개

Firstep은 사이드프로젝트를 처음 시작하는 1인 개발자가 아이디어를 실제 서비스로 완성할 수 있도록 돕는 AI 기반 프로젝트 관리 웹앱이다.

## 타겟

- 사이드프로젝트를 처음 혼자 시작하는 개발자
- 아이디어는 있지만 기획/개발 순서를 모르는 사람
- 1인 개발 시 기술 스택 선택이 막막한 사람

## 해결하는 문제

1. "뭐부터 해야 하지?" → 기획, 설계, 개발, 배포 순서를 몰라 시작을 못 함
2. "혼자 하다 흐지부지" → 동기부여가 떨어지고 진행이 멈춤
3. "서버, DB, 배포 뭐 써야 해?" → 1인 개발에 맞는 기술 스택 정보 부족

## 회원 관리

- 구글 로그인만 지원 (개인정보 처리 부담 최소화)
- 추후 GitHub 로그인 추가 검토

## 1단계 MVP

### 온보딩 플로우

구글 로그인 → 스택 온보딩(선택, 스킵 가능) → 아이디어 있어요/없어요 선택 → AI 대화로 기획 구체화 → 로드맵 생성

온보딩 이탈 방지:
- 스택 선택은 선택사항
- 스킵해도 서비스 이용 가능
- 로드맵 생성 후 넛지: "스택을 추가하면 더 정확한 추천을 받을 수 있어요"

### AI 기획 구체화 + 로드맵 생성

두 가지 진입 케이스:
- 아이디어 있는 경우 → 5지선다 질문으로 구체화 → 로드맵 생성
- 아이디어 없는 경우 → 페인포인트 발굴 질문부터 시작 → 대화를 통해 아이디어 도출 후 로드맵 생성

AI 대화 한도:
- 토큰 기반 제한
- UI는 게이지 바 + 숫자 병행 표시
- 무료/PRO 플랜 간 차등 제공 (수치는 베타 후 결정)
- 기획 대화 단계에서만 AI 대화 가능
- 로드맵 확정 후에는 막힌 스텝 힌트 용도로만 사용

### AI 운영 정책

**토큰 한도 산정 방식**
- Claude API 실사용 단가(input/output per-token) 기반으로 원가를 역산해 한도 결정
- 무료 한도는 "유저당 월 원가 ≤ X원" 기준으로 상한 설정
- PRO 한도는 무료 × 5를 기본으로 하되, 실제 소모량 분포 보고 조정

**남용 방지**
- 이메일당 1계정 (구글 OAuth 기반 중복 가입 차단)
- 동일 IP에서 연속 로드맵 생성 시 쿨다운 (예: 10분)
- 비정상 프롬프트 필터: 과도한 길이, 반복 입력, 금지 패턴 차단
- 일일 최대 요청 횟수 상한 (토큰 한도와 별개로 안전장치)

**부분 실패 시 토큰 환불**
- 스트림 중단, 형식 오류, API 에러 시 이미 차감한 토큰 자동 복구
- 환불 내역은 유저에게 투명하게 노출 ("생성 실패로 토큰 N개 복구됨")

**프롬프트 인젝션 방어**
- 유저 입력은 시스템 프롬프트와 격리된 영역에 삽입
- 시스템 프롬프트를 덮어쓰려는 시도 감지 시 요청 차단
- 로드맵 생성 결과를 Zod 등 스키마로 검증해 구조 깨뜨리기 공격 방어

### 로드맵 품질 관리

**품질 피드백 루프**
- 블록/투두별 👍 / 👎 버튼 제공 → 피드백 데이터 축적
- 👎 클릭 시 사유 선택 (너무 큼 / 너무 모호함 / 현실성 없음 / 스택 안 맞음)
- 누적 피드백을 주기적으로 프롬프트 튜닝 재료로 활용

**스택 화이트리스트**
- AI가 추천 가능한 라이브러리/프레임워크/서비스 목록을 서버에서 관리
- 2025~2026년 기준 유지보수 활발하고 1인 개발에 적합한 것 위주로 큐레이션
- 화이트리스트 외 추천이 감지되면 검증 프롬프트로 재질문하거나 대체안 제시

**재생성 가드레일**
- 블록 단위 원클릭 재생성 트리거: "너무 큼", "모호함", "현실성 낮음" 버튼 → 사유 포함해 N/5 토큰으로 재생성
- 전체 재생성은 기존대로 N 토큰, 단 **연속 재생성 방지** (같은 프롬프트 3회 반복 시 경고)

**로드맵 템플릿 시드**
- 자주 나오는 프로젝트 유형별 기준 템플릿을 Few-shot 예시로 시스템 프롬프트에 주입
  - 웹앱 / 랜딩 페이지 / 챗봇 / 크롬 확장 / 모바일(PWA) 등
- 템플릿은 품질 하한선 역할만 하며, 유저 아이디어에 맞게 변주되어야 함

### Claude API 장애 대응

장애 유형별로 구분해 UX와 내부 처리 로직을 분리:

**1. 일시적 실패 (Rate Limit / 네트워크 hiccup)**
- 서버에서 지수 백오프 자동 재시도 (1s → 2s → 4s, 최대 3회)
- 유저 화면: 로더 유지 + "AI가 생각 중이에요... (재시도 중)"
- 최종 실패 시 "잠시 혼잡해요. 다시 시도" 버튼, 토큰 차감 없음

**2. 장기 장애 (Claude API 다운)**
- Circuit Breaker: 연속 실패 N회 감지 시 1분간 요청 차단 (비용 폭주 방지)
- 요청을 DB `pending` 큐에 저장 → 복구 감지 시 워커가 자동 처리
- 복구 완료 시 **이메일(Resend) + 앱 내 토스트 둘 다** 발송
- 토큰은 최종 성공 시점에만 차감 (pending 상태에선 예약만)

**3. 부분 실패 (스트리밍 중 끊김)**
- 스트림 청크 단위로 DB 저장 → 끊겨도 중간까지 보존
- 유저 선택지: "이어서 생성" (남은 블록만 재요청, 토큰 절약) / "직접 편집" (AI 없이 완성)

**4. 응답 형식 오류 (JSON 파싱 실패 / 할루시네이션)**
- Zod 등으로 응답 스키마 검증 → 실패 시 1회 자동 재요청
- 계속 실패 시 유저에게 "AI가 이상한 답변을 했어요, 다시 시도할까요?" + 토큰 복구
- 실패한 원본 응답은 로그로 보관 (프롬프트 튜닝 재료)

**5. 토큰 소진 (장애 아니지만 동일 플로우)**
- 잔여 20% 이하 시 상단 배너로 사전 경고
- 소진 시 "다음 달까지 대기 or PRO 업그레이드" 안내
- 조회/체크 등 토큰 무관 기능은 계속 사용 가능

**공통 안전장치**
- Sentry로 5xx / 파싱 실패 수집, 임계치 초과 시 운영자 알림
- Redis/DB 트랜잭션으로 토큰 차감 ↔ 실패 시 롤백 원자성 보장
- Fallback 모델 다운그레이드(Sonnet → Haiku)는 **채택하지 않음** (MVP 단계에서 품질 저하 리스크가 더 큼)

### 로드맵 구조

블록(큰 단위) > 투두(작은 단위)

블록 커스터마이징:
- 순서 드래그 앤 드롭
- 블록 추가/삭제/이름 수정 (토큰 소모 없음)
- AI로 블록 재생성 → 토큰 N/5 소모

투두 기준:
- 로컬에서 실행해서 눈으로 확인할 수 있는 단위
- 30분~2시간 안에 끝낼 수 있는 크기
- 순서 강제 없음
- 투두 추가/삭제 가능

### 빌드 로그 감성 대시보드

- DONE/RUN/TODO 상태 표시
- 전체 진행률 시각화
- 연속 개발일 스트릭
- 오늘 완료한 투두 수
- AI 토큰 잔여량 게이지

### 이메일 넛지

- 3일 연속 미접속 시 자동 이메일 발송
- 수신 거부 옵션 제공

### 데이터 처리

- 프로젝트 삭제 시 소프트 딜리트
- 계정 탈퇴 시 데이터 파기 기한: **30일 이내 완전 삭제** (법적 보관 의무 항목 제외)
- 탈퇴 신청 후 30일간은 복구 요청 가능 (복구 요청 UI는 마이페이지 또는 재로그인 시 제공)

### 데이터 내보내기 (Export)

유저 락인 우려를 낮추고 신뢰를 확보하기 위해 프로젝트 데이터 반출 지원:
- **Markdown**: 로드맵 전체를 한 파일로 (블록/투두 체크박스 포함)
- **GitHub Issues**: 블록 = Milestone, 투두 = Issue로 일괄 생성 (OAuth 연동 전까지는 JSON 템플릿 제공)
- **Notion**: 워크스페이스 API 연동 또는 import 가능한 Markdown/CSV
- 내보내기는 무료 플랜 포함 기본 기능 (PRO 유도 목적 아님)

### 공유 / 소셜

- **로드맵 공개 링크**: 프로젝트별 공개 URL 생성 (예: `firstep.io/p/:slug`)
  - 공개 / 비공개 토글, 기본값 비공개
  - 공개 시 읽기 전용 뷰 (투두 체크 불가, 진행률만 노출)
- **SEO 유입**: 공개 로드맵은 검색엔진 인덱싱 허용 → "어떻게 사이드프로젝트 시작하지" 검색 유입 장치
- **소셜프루프**: 랜딩에 "최근 시작된 프로젝트" 공개 목록 (익명화 옵션)
- **빌드 인 퍼블릭 공유**: 투두 완료 시 "진행 상황 공유" 버튼 → X(트위터)/링크드인용 이미지 카드 생성 (2단계)

### 프로젝트 상태 정의

- active: 진행 중
- completed: 완료
- locked: 플랜 다운그레이드로 잠금 (읽기 전용)

### 플랜 다운그레이드 정책 (PRO → 무료)

- 무료 한도(프로젝트 1개) 초과분은 locked 처리
- 잠긴 프로젝트는 읽기 전용 유지
- PRO 재구독 시 즉시 복구

## 2단계

- 게임화 시스템 (레벨, 퀘스트, 뱃지, 코인)
- 브라우저 익스텐션 (Chrome)

## 3단계 (검토)

- 투두 완료 시 AI 커밋 메시지 자동 생성/제안

## 수익 모델

### 무료 플랜
- 프로젝트 1개, AI 토큰 월 N, 기본 대시보드

### 유료 플랜 (PRO) - 월 3,000~5,000원
- 프로젝트 무제한, AI 토큰 월 N×5, 상세 통계, PDF 업로드

### 코인 시스템
- 퀘스트 달성 시 코인 적립 → 유료 플랜 한시적 구매 가능

## 결제 · 법적 컴플라이언스 (검토 / 결정 필요)

유료 플랜 론칭 전 반드시 확정해야 하는 항목. **아래는 제안이며, 항목별 결정이 필요함**:

### 결제 인프라 후보
- **토스페이먼츠**: 한국 시장 안정적, 정기결제 지원, 수수료 중간
- **포트원(구 아임포트)**: 여러 PG 통합 관리, 국내 카드 호환성 우수
- **Stripe**: 해외 확장 고려 시 유리하나 한국 카드 이슈 존재
- → 국내 타겟이면 토스페이먼츠 / 포트원 중 택일 권장

### 필수 약관
- 이용약관
- 개인정보처리방침
- 결제 약관 (정기결제 명시)
- 환불 정책 — 전자상거래법 상 7일 청약 철회권, 디지털 콘텐츠 소비 개시 시 예외 조항 명시 필요

### 개인정보 처리
- 구글 OAuth로 수집하는 항목(이메일 / 프로필) 명시적 동의
- **해외 이전 동의**: Supabase 리전, Claude API(Anthropic) 처리가 해외에서 이루어지면 별도 고지·동의 필요
- 14세 미만 가입 처리 정책 (한국 개인정보보호법)

### 데이터 파기
- 계정 탈퇴 시 **30일 이내 완전 삭제** (상위 "데이터 처리" 섹션과 일치)
- 법적 보관 의무가 있는 항목(결제 기록 등)은 관련 법에 따라 별도 기간 보관

- [ ] 환불 정책의 디지털 콘텐츠 예외 조항 적용 범위
