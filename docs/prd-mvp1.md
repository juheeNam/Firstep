## 2. MVP 1단계 범위

### 2.1 포함 항목
- **인증**: 구글 OAuth 로그인 + 스택 온보딩(선택사항)
- **AI 로드맵 생성**: 기획 대화 → 블록/투두 구조 생성
- **투두 관리 + 대시보드**: 체크·편집·드래그, 진행률·스트릭 시각화

### 2.2 제외 항목
아래는 기획서에 있으나 이번 MVP 1단계 스코프에서 제외:
- AI 힌트 기능 (유저 코드 컨텍스트 없이는 정확도 낮음 → 영구 제외)
- **기획서·PRD 파일(Markdown/PDF) 첨부로 로드맵 생성** — 2단계+ PRO 기능 (파싱 복잡도 큼)
- 이메일 넛지
- 결제 / 플랜 관리 (초기 무제한, 베타 후 1개 제한 도입)
- 공유 링크 / 데이터 내보내기
- 삭제된 프로젝트 복구 UI (소프트 딜리트 자체는 지원, UI는 2단계+)
- 기획 대화 이력 DB 저장 / 복구 (세션 스토리지만 사용)
- Claude API 고도화 장애 대응(Circuit Breaker, 큐잉)
- 게임화(레벨/퀘스트/뱃지/코인) — 2단계
- 브라우저 익스텐션 — 2단계
- AI 커밋 메시지 — 3단계

## 3. 타겟 사용자

- 사이드프로젝트를 처음 시작하는 1인 개발자
- 아이디어는 있지만 기획/개발 순서를 모르는 사람
- 1인 개발 시 기술 스택 선택이 막막한 사람

## 4. 기능 요구사항

### 4.1 구글 로그인

**유저 스토리**
> 사용자는 구글 계정으로 로그인하여 Firstep을 이용할 수 있다.

**요구사항**
- Supabase Auth + Google OAuth 연동
- OAuth 스코프: 이메일 + 기본 프로필 (최소 수집)
- 최초 로그인 시 자동 회원가입
- 로그아웃 기능 제공

**인수 기준**
- [ ] 랜딩에서 "구글로 시작하기" 클릭 시 OAuth 플로우 진입
- [ ] OAuth 성공 시 최초 로그인 → `/onboarding`, 재로그인 → `/dashboard`
- [ ] 세션 만료 시 로그인 페이지로 자동 리다이렉트
- [ ] 로그아웃 후 보호된 경로 접근 시 로그인 페이지로 리다이렉트

### 4.2 스택 온보딩 (선택사항)

**유저 스토리**
> 사용자는 자신이 써본/써보고 싶은 기술 스택을 선택해 AI 로드맵을 개인화한다. 스킵해도 서비스 이용 가능.

**요구사항**
- 5개 카테고리, 각 10개 옵션, 4상태 (미선택 / 써봤어요 / 써보고싶어요 / 관심없어요)
- 스킵 버튼 제공 → 미선택 시 AI가 보편적인 스택 추천
- 마이페이지에서 언제든 재편집
- 로드맵 생성 후 스킵 유저에게 넛지 배너 노출: "스택을 추가하면 더 정확한 추천을 받을 수 있어요"

**카테고리 및 옵션**

| 카테고리 | 옵션 (10개) |
|---|---|
| 프론트엔드 | React, Next.js, Vue, Nuxt, Svelte/SvelteKit, Angular, Solid, Astro, Remix, React Native |
| 백엔드 | Node.js(Express), NestJS, Next.js Route Handler, Django, FastAPI, Spring Boot, Rails, Go(Gin/Echo), Laravel, Hono |
| DB | PostgreSQL, MySQL, SQLite, MongoDB, Redis, Supabase, Firebase, PlanetScale, Turso, Neon |
| 인프라 | Vercel, Netlify, AWS, GCP, Cloudflare Pages/Workers, Railway, Fly.io, Render, Heroku, Docker |
| 보안/네트워크 | Supabase Auth, Auth.js(NextAuth), Clerk, Firebase Auth, OAuth 2.0/OIDC, JWT, Cloudflare(CDN/WAF), Nginx, Let's Encrypt(HTTPS), Linux |

**인수 기준**
- [ ] 각 옵션 클릭 시 4상태 순환 (미선택 → 써봤어요 → 써보고싶어요 → 관심없어요 → 미선택)
- [ ] 카테고리별 시각적 구분
- [ ] "나중에 하기" 버튼으로 스킵 가능
- [ ] 저장 시 `/dashboard`로 이동, 스킵 시에도 `/dashboard`로 이동
- [ ] 마이페이지 재편집 시 기존 선택 상태 유지

### 4.3 AI 기획 대화

**유저 스토리**
> 사용자는 본인의 아이디어 구체화 수준(readiness)에 맞춰 진입 플로우를 선택하고, AI와의 대화를 통해 로드맵 생성에 필요한 정보를 제공한다.

#### 4.3.1 진입점 분기 (3갈래)

프로젝트 시작 시 유저에게 다음 3가지 중 하나 선택:

| 레벨 | 라벨 | 대상 유저 | 플로우 |
|---|---|---|---|
| 1 | 🌱 "아직 아이디어가 없어요" | 막연한 관심만 있음 | 페인포인트 발굴 (4.3.2) |
| 2 | 💡 "대략적 아이디어가 있어요" | 컨셉 정도 잡힘 | 5지선다 + 자유 텍스트 (4.3.3) |
| 3 | 📋 "구체적 기획이 있어요" | 상세 계획/기획서 보유 | 자유 서술 + AI 후속 (4.3.4) |

- 선택 화면 상단에 간단한 설명 카피
- 언제든 뒤로가기 / 다른 레벨로 전환 가능

#### 4.3.2 레벨 1: 페인포인트 발굴 플로우
- **고정 3개 질문** 순차 노출:
  1. "요즘 일상에서 제일 귀찮거나 불편한 게 뭐예요?"
  2. "주변에서 자주 듣는 불평이 뭐예요?"
  3. "반복적으로 하는 일 중에 자동화하고 싶은 게 있나요?"
- 유저 응답 기반 AI가 동적 후속 질문 **2~4턴** 생성
- 수렴된 아이디어를 AI가 요약 → 유저 확인 → 로드맵 생성 단계로 이동

#### 4.3.3 레벨 2: 5지선다 + 자유 텍스트 플로우

총 **5턴** (자유 텍스트 1 + 객관식 4), 일관 순서, 조건부 분기 없음.
Q2 / Q3 / Q5는 단일 선택, **Q4(외부 통합)는 복수 선택**.

**턴 순서**

| # | 유형 | 질문 | 선택지 / 입력 |
|---|---|---|---|
| 1 | 자유 텍스트 | "만들고 싶은 서비스를 한 줄로 설명해주세요" | 예: "헬스장 운동 기록을 친구들과 공유하는 앱" |
| 2 | 단일 선택 (5지선다) | "어떤 형태의 프로젝트인가요?" | 웹앱 / 모바일 앱 / 브라우저 확장 / 데스크톱·CLI / 랜딩 페이지 |
| 3 | 단일 선택 (5지선다) | "사용자 인증/개인화는 어디까지 필요해요?" | 필요 없음 / 로그인만 / 유저별 데이터 / 역할·권한 분리 / 결제까지 |
| 4 | **복수 선택** | "어떤 외부 통합이 필요해요? (해당되는 것 모두 선택)" | 결제 / 이메일·알림 / AI(LLM) / 외부 API 연동 / 없음 / 잘 모르겠어요 |
| 5 | 단일 선택 (5지선다) | "누구한테 보여줄 계획인가요?" | 나만 사용 / 지인 공유 / 공개 베타 / 정식 출시 / 수익화 목표 |

- **Q4 복수 선택 규칙**:
  - "없음" 또는 "잘 모르겠어요" 선택 시 다른 항목 해제 (배타적)
  - 최소 1개 이상 선택해야 다음 단계 진행
  - Q1 자유 텍스트에 언급되지 않은 통합을 유저가 보완해서 선택 가능 (AI 추론 오차 방지)
- "잘 모르겠어요"는 Q4에만 제공 (외부 통합은 초보자가 어려울 수 있음). Q2/Q3/Q5는 5개 선택지로 충분해 별도 추가 안 함.
- "기타(자유입력)"는 없음 — 흐름 깨지고 AI 해석 품질 저하 위험.
- 완료 시 AI가 응답을 종합해 기획 요약 제시 → 유저 확인 → 로드맵 생성 단계로 이동.

#### 4.3.4 레벨 3: 자유 서술 + AI 후속 플로우

구체 기획이 있는 유저용 심화 플로우:

- **큰 textarea** 1개 제공, 자유 서술 입력 (예: "헬스장 운동 기록 공유 앱. 주요 기능은 운동 일지 기록, 친구 팔로우, 주간 리포트. 타겟은 20-30대 헬스 초보자...")
- 입력 최소 권장 분량: **100자** (그 이하면 "더 자세히 알려주시면 더 정확한 로드맵을 만들 수 있어요" 안내)
- 제출 후 AI가 부족한 정보를 식별해 **2~4턴 자유 대화**로 추가 질문 (5지선다 X)
  - 예시 AI 질문: "인증은 어떻게 처리하실 계획인가요?" / "결제 기능도 포함되나요?"
- AI가 정보 충분하다고 판단하거나 유저가 "이제 로드맵 만들기" 버튼 클릭 시 종료
- AI가 정리된 기획 요약 제시 → 유저 확인 → 로드맵 생성 단계로 이동

> 📎 **파일(Markdown/PDF) 첨부로 로드맵 생성**은 MVP 1단계에서 제외 (2단계+ PRO 기능). 파싱 복잡도가 높고, 타겟 유저 비율이 낮음.

#### 4.3.5 공통: 기획 요약 & 제목 확정

3개 레벨 모두 대화 완료 후 공통으로 거치는 단계:

- AI가 대화 내용을 종합해 **프로젝트 제목(`title`) + 기획 요약(`idea_summary`)** 두 가지를 제안
- 유저는 요약 화면에서 둘 다 **인라인 편집 가능** (제목 필수 입력, 빈 값일 시 다음 단계 블록)
- 요약 내용이 마음에 들지 않으면 **"다시 대화하기" 버튼** → 마지막 턴으로 돌아가 이어서 수정 (기존 대화 이력 유지, 토큰은 추가 턴에 대해서만 소모)
- "이 내용으로 로드맵 만들기" 버튼 클릭 시 로드맵 규모 선택(4.4.1)으로 이동

#### 4.3.6 AI 대화 토큰 정책
- 토큰 기반 제한, UI 게이지 바 + 숫자 병행 표시
- **기획 대화 단계에서만** AI 토큰 소모 (3개 플로우 모두 해당)
- 로드맵 확정 후에는 AI 대화 불가
- 무료/PRO 차등 (구체 수치는 베타 후 결정)

#### 4.3.7 대화 상태 관리
- 기획 대화 중 입력·선택 내용은 **세션 스토리지**에 보관 (MVP 단순화)
- 브라우저 닫거나 새로고침 시 처음부터 다시 시작 (DB 저장 및 복구는 2단계+)
- 대화 이력 DB 저장이 필요한 2단계 기능은 별도 PRD로 분리

**인수 기준**
- [ ] 진입점 선택 화면에서 3개 레벨 중 하나 선택 가능
- [ ] 레벨 간 전환(뒤로가기) 시 기존 입력 초기화 안내
- [ ] 레벨 1: 고정 3개 질문 순차 노출 후 AI 동적 후속 2~4턴
- [ ] 레벨 2: 자유 텍스트(Q1) → 객관식 4개(Q2~Q5) 순서, 총 5턴 고정
- [ ] 레벨 2: Q2/Q3/Q5는 단일 선택, Q4는 복수 선택
- [ ] 레벨 2: Q4 선택지에 "없음", "잘 모르겠어요" 포함 (두 옵션은 배타적 — 선택 시 다른 항목 해제)
- [ ] 레벨 2: Q4 최소 1개 선택해야 다음 단계 진행
- [ ] 레벨 3: textarea 입력 후 AI 후속 질문 2~4턴, "이제 로드맵 만들기" 버튼으로 조기 종료 가능
- [ ] 레벨 3: 100자 미만 입력 시 분량 안내 배너 표시 (블로킹 아님)
- [ ] 요약 단계: AI가 `title` + `idea_summary` 제안, 유저가 인라인 편집 가능 (제목 필수)
- [ ] 요약 단계: "다시 대화하기" 버튼으로 마지막 턴 복귀 가능, 기존 이력 유지
- [ ] 대화 상태는 세션 스토리지만 사용, 브라우저 닫거나 새로고침 시 처음부터 시작
- [ ] 모든 플로우 종료 시 "이 내용으로 로드맵 만들기" CTA
- [ ] 토큰 잔여량 실시간 UI 반영, 소진 시 생성 불가 안내
- [ ] 프롬프트 인젝션 방어: 유저 입력은 시스템 프롬프트와 격리된 영역에 삽입
- [ ] 프로젝트 저장 시 `entry_level`(1/2/3) + `intent_data`(JSONB) 기록

### 4.4 로드맵 생성 & 편집

**유저 스토리**
> 사용자는 확정된 기획을 바탕으로 블록/투두 로드맵을 생성하고, 필요에 따라 순서 조정·추가·삭제·재생성할 수 있다.

#### 4.4.1 로드맵 규모 선택
로드맵 생성 직전, 3가지 옵션 중 선택:

| 옵션 | 블록 수 | 투두 수 (전체) | 대상 예시 |
|---|---|---|---|
| **자동 (AI 판단)** — *기본값 추천* | 동적 | 동적 | 유저가 맡길 때 |
| **기본** | 4-7 | 15-30 | 블로그·간단 CRUD·랜딩페이지·단일 기능 도구 |
| **상세** | 8-15 | 40-80 | 인증·결제·AI·대시보드 등 복합 모듈 포함 서비스 |

#### 4.4.2 로드맵 생성
- Claude API **스트리밍**으로 블록/투두 순차 생성
- 생성 결과는 Zod 스키마로 서버에서 검증
- 검증 실패 시 1회 자동 재요청, 계속 실패 시 **토큰 환불** + 유저에게 재시도 안내
- **생성 중 취소**(유저 뒤로가기 / 취소 버튼) 시 서버 스트림 중단 + **토큰 전액 환불** + 부분 결과 저장하지 않음
- 생성 완료 후 로드맵 뷰로 자동 이동

#### 4.4.3 AI 프롬프트 가이드 (핵심 요구사항)
```
- 선택된 규모 옵션(기본/상세/자동)에 맞춰 블록 수 결정
- 각 투두는 "개발자가 직접 코드를 작성할 때 1~3시간(평균 2시간)" 내 완료 가능한 크기
  * 과도한 쪼갬 금지: "로그인 버튼 생성"(X) → "로그인 페이지 UI + Supabase 연동"(O)
  * 로컬에서 실행해 눈으로 확인 가능한 단위
- 스택 온보딩 입력 반영: "써봤어요" > "써보고싶어요" > 중립 순으로 우선 추천
- 화이트리스트 외 라이브러리 추천 시 검증 프롬프트로 재질문
- 결과는 정의된 JSON 스키마를 엄격히 준수
```

> UI에는 시간 정보(1~3시간)를 **노출하지 않음** — 유저 페이스에 따라 실제 소요 시간이 다를 수 있으므로 AI 프롬프트 가이드용으로만 사용.

#### 4.4.4 블록 편집
- 드래그 앤 드롭으로 블록 순서 변경 (`seq` 업데이트)
- 블록 추가 / 삭제 / 이름 수정 (토큰 소모 없음)
- 블록 단위 AI 재생성 (토큰 **N/5** 소모)
- 로드맵 전체 재생성 (토큰 **N** 소모, 최초 생성과 동일)

#### 4.4.5 프로젝트 정보 편집 (로드맵 뷰 상단)
- 프로젝트 **제목** 인라인 편집 가능 (AI가 제안한 기본값 위에 덮어쓰기)
- 제목 빈 값 저장 불가

**인수 기준**
- [ ] 로드맵 규모 선택 화면 제공 (자동/기본/상세)
- [ ] 스트리밍 생성 중 로더 + "AI가 로드맵을 만드는 중이에요" + 취소 버튼 표시
- [ ] 생성 중 취소 시 스트림 중단 + 토큰 전액 환불 + 부분 결과 저장 안 함
- [ ] 생성 완료 후 블록/투두 트리 뷰 렌더링
- [ ] 블록 드래그 앤 드롭 시 `seq` 필드 업데이트, 새로고침 후에도 순서 유지
- [ ] 블록 우측 "AI로 다시 짜기 ✨" 버튼, 클릭 시 토큰 소모 확인 모달 → 승인 시 해당 블록만 재생성
- [ ] 전체 재생성 버튼 (토큰 N 소모 확인 모달)
- [ ] 스키마 검증 실패 시 1회 자동 재시도, 최종 실패 시 토큰 환불 및 유저 알림
- [ ] 로드맵 뷰 상단에서 프로젝트 제목 인라인 편집 가능, 빈 값 저장 차단

### 4.5 투두 관리

**유저 스토리**
> 사용자는 투두를 체크/해제, 순서 변경, 직접 편집, 추가/삭제하며 진행 상황을 관리한다.

**요구사항**
- 체크박스 토글 (완료 ↔ 미완료) — **체크 해제 허용**
- 드래그 앤 드롭으로 같은 블록 내 투두 순서 변경 (`seq` 업데이트)
- 투두 내용 **직접 편집** (AI 없이, 인라인 편집)
- 투두 추가 / 삭제
- 순서 강제 없음 (뒤 투두 먼저 체크해도 막지 않음)

**인수 기준**
- [ ] 투두 체크박스 클릭 시 상태 토글 + DB 즉시 반영
- [ ] 체크 해제 동작 정상
- [ ] 투두 드래그로 블록 내 `seq` 변경
- [ ] 투두 텍스트 더블클릭 또는 편집 아이콘으로 인라인 편집
- [ ] 투두 추가 버튼으로 해당 블록 하단에 새 투두 생성
- [ ] 투두 삭제 시 확인 모달

### 4.6 대시보드

**유저 스토리**
> 사용자는 프로젝트 진행 상황을 시각적으로 확인하며 동기부여를 받는다.

**요구사항**
- **전체 진행률**: 투두 기준 = `(완료 투두 수 / 전체 투두 수) × 100%`
- **연속 개발일(스트릭)**: 하루 **1개 이상 투두 완료** 시 +1일, 0개인 날은 리셋
- **오늘 완료한 투두 수**: 유저 브라우저 로컬 타임존 기준
- **AI 토큰 잔여량**: 게이지 바 + 숫자, 20% 이하 시 경고 배너
- **블록 상태 배지**: `DONE`(전체 투두 완료) / `RUN`(일부 완료) / `TODO`(전체 미완료)

**인수 기준**
- [ ] 대시보드 진입 시 진행률 %가 정확히 계산되어 표시
- [ ] 스트릭 카드 표시 (오늘 첫 투두 완료 전/후로 업데이트)
- [ ] 오늘 완료한 투두 수 카드
- [ ] 토큰 잔여량 게이지 (20% 이하 경고 배너)
- [ ] 블록별 상태 배지 (DONE/RUN/TODO)

### 4.7 프로젝트 관리

**유저 스토리**
> 사용자는 여러 프로젝트를 리스트로 관리하고, 필요 시 삭제하거나 완료 상태로 전환할 수 있다.

#### 4.7.1 프로젝트 리스트
- 대시보드 또는 별도 경로(`/projects`)에 **카드 뷰**로 노출
- 각 카드: 프로젝트 제목 / 진행률(%) / 상태 배지(active · completed · locked) / 최근 수정일
- 기본 정렬: 최근 수정일 내림차순
- 우측 상단 "+ 새 프로젝트" 버튼 → 진입점 선택(4.3.1)으로 이동

#### 4.7.2 프로젝트 삭제
- 카드 ⋯ 메뉴 또는 로드맵 뷰 상단 메뉴에서 "삭제" 옵션
- 삭제 확인 모달 ("정말 삭제하시겠어요? 복구는 현재 지원되지 않아요.")
- 확인 시 **소프트 딜리트** (`deleted_at` 기록, 리스트에서 제외)
- 복구 UI는 MVP 제외 (2단계+)

#### 4.7.3 프로젝트 완료 전환
- 전체 투두 완료 시 대시보드/로드맵 뷰에 **"🎉 프로젝트 완료하기" CTA** 노출
- 유저 수동 클릭 → `status`를 `active` → `completed`로 전환
- 자동 전환 안 함 (유저가 투두 추가·수정할 여지를 남겨야 하므로)
- 완료된 프로젝트도 편집 가능하지만, 완료 상태 해제를 원하면 "다시 진행하기" 버튼으로 `active` 복귀

**인수 기준**
- [ ] 프로젝트 리스트 카드 뷰 렌더링 (제목 / 진행률 / 상태 / 최근 수정일)
- [ ] 기본 정렬: 최근 수정일 내림차순
- [ ] "+ 새 프로젝트" 버튼 → 진입점 선택 화면
- [ ] 카드 ⋯ 메뉴에 "삭제" 옵션, 확인 모달 후 소프트 딜리트
- [ ] 삭제된 프로젝트는 리스트에 노출되지 않음 (`deleted_at IS NOT NULL` 필터)
- [ ] 전체 투두 완료 시 "프로젝트 완료하기" CTA 노출, 클릭 시 `status = completed`
- [ ] 완료된 프로젝트도 편집 가능, "다시 진행하기" 버튼으로 `active` 복귀

### 4.8 빈 상태 (Empty State)

**유저 스토리**
> 첫 가입 유저는 프로젝트가 없을 때 명확한 다음 액션을 안내받는다.

**요구사항**
- 대시보드 진입 시 프로젝트 0개면 "첫 프로젝트를 시작해볼까요?" CTA
- CTA 클릭 시 진입점 선택 화면으로 이동

**인수 기준**
- [ ] 프로젝트 0개 시 빈 상태 화면 렌더링
- [ ] CTA 클릭 시 AI 기획 대화 진입점(4.3.1)으로 이동

## 6. 데이터 모델 (초안)

### 6.1 주요 테이블

```sql
-- Supabase auth.users 를 기본으로 사용, 프로필은 별도 테이블
profiles
  id uuid PK (auth.users.id 참조)
  email text
  full_name text
  avatar_url text
  created_at timestamptz

user_stacks
  id uuid PK
  user_id uuid FK → profiles.id
  category text  -- 'frontend' | 'backend' | 'db' | 'infra' | 'security'
  tech_name text
  status text   -- 'used' | 'want' | 'not_interested'
  updated_at timestamptz
  UNIQUE (user_id, category, tech_name)

projects
  id uuid PK
  user_id uuid FK → profiles.id
  title text                 -- AI 제안 기본값, 유저 편집 가능, 빈 값 금지
  idea_summary text          -- AI 요약 결과
  entry_level smallint       -- 1 | 2 | 3 (진입 플로우)
  intent_data jsonb          -- {q1_text, q2_single, q3_single, q4_multi:[], q5_single, level3_raw_text, ai_followups:[]}
  size_option text           -- 'auto' | 'basic' | 'detailed'
  status text                -- 'active' | 'completed' | 'locked'
  created_at timestamptz
  updated_at timestamptz
  deleted_at timestamptz     -- soft delete

blocks
  id uuid PK
  project_id uuid FK → projects.id
  name text
  seq integer               -- 프로젝트 내 정렬 순서
  created_at timestamptz
  updated_at timestamptz

todos
  id uuid PK
  block_id uuid FK → blocks.id
  content text
  is_done boolean
  seq integer               -- 블록 내 정렬 순서
  completed_at timestamptz  -- nullable
  created_at timestamptz
  updated_at timestamptz

ai_usage                    -- 토큰 회계
  id uuid PK
  user_id uuid FK → profiles.id
  project_id uuid FK → projects.id (nullable)
  action text               -- 'roadmap_generate' | 'block_regenerate' | 'chat' | 'refund_cancel' | 'refund_error'
  tokens_delta integer      -- 양수=차감, 음수=환불
  created_at timestamptz
```

### 6.2 RLS 정책 (요약)
- `profiles`: 본인만 SELECT/UPDATE
- `user_stacks`, `projects`, `ai_usage`: `user_id = auth.uid()` 격리
- `blocks`, `todos`: 조인된 project의 user_id 기준 간접 격리

## 9. 가정 및 유보 항목

### 9.1 가정 (명시 지시 없어 기본값으로 진행)
- OAuth 스코프: 이메일 + 기본 프로필 (최소 수집)

- 스트릭 리셋: 하루라도 투두 0개면 다음 날 0으로 리셋
- **언어**: MVP는 한국어 전용 (다국어 지원은 별도 논의)
- **기획 대화 이력**: 세션 스토리지만 사용, 브라우저 닫으면 처음부터 (DB 저장·복구는 2단계+)
- **로그아웃 버튼 위치**: 헤더 우측 프로필 드롭다운 가정 (디자인 시점에 확정)
- **404 / 500 에러 페이지**: 디자인 단계에서 확정
